# Playbook

이 플레이북은 ACE(Agentic Context Engineering) 프레임워크를 통해 관리되는 `gcs-front` 프로젝트의 진화하는 지식 베이스입니다.

## 전략 (Strategies)
- **API 연동과 디자인 시스템 조화**: 기존의 디자인 시스템 컴포넌트를 활용하여 API 데이터를 표시함으로써 일관된 UX를 유지함.
- **인증 상태 중앙 관리**: `AuthContext`를 통해 어플리케이션 전체의 인증 상태를 관리하고, `ProtectedRoute` 컴포넌트로 접근 제어를 단순화함.
- **크로스 도메인 세션 관리**: 프론트엔드와 백엔드가 서로 다른 도메인/포트에 있을 때 쿠키 기반 세션을 유지하기 위한 전략 수립.
- **CORS 오류 배후의 500 에러**: 브라우저에 CORS 오류가 뜰 때 실제로는 백엔드의 `500 Internal Server Error`가 원인일 수 있습니다. 서버가 에러로 죽으면서 CORS 헤더를 응답에 포함하지 못해 발생하는 현상이므로 반드시 백엔드 로그를 먼저 확인해야 합니다.
- **프리미엄 관리자 UI 설계**: 복잡한 설정은 인라인 편집보다 모달(Modal) 기반의 CRUD 인터페이스가 사용자 경험이 더 뛰어납니다. HTTP 메소드별 색상 구분(GET-녹색, POST-황색 등)을 통해 시각적 가독성을 확보합니다.
- **중앙 집중식 게이트키퍼(Gatekeeper) 전략**: `/` 경로를 단순한 랜딩이 아닌 상태 판별기(Router)로 활용하여 [Auth -> Consent -> Role] 순으로 유저 동선을 엄격히 통제합니다. 이는 폐쇄형 B2B/LMS 서비스에서 보안과 온보딩 경험을 동시에 잡는 효과적인 방식입니다.
- **원자적 상태 동기화 (Atomic Auth Sync)**: 약관 동의나 프로필 수정 등 유저 상태를 변경하는 작업을 완료한 직후에는 반드시 `checkAuth()`와 같은 전역 인증 갱신 함수를 호출해야 합니다. 서버 데이터와 로컬 컨텍스트 사이의 시차(Lag)는 무한 리다이렉트 루프나 잘못된 접근 제어의 주원인이 됩니다.

- **치명적 레이아웃의 안전장치 (Layout Isolation)**: 로그인/랜딩 페이지처럼 디자인의 정밀도가 중요한 화면에서 외부 CSS 프로젝트(전역 스타일 등)의 간섭으로 정렬이 깨질 경우, Tailwind 유틸리티에만 의존하지 말고 **인라인 스타일(Inline Styles)**을 사용하여 레이아웃 주도권을 확실히 가져와야 합니다.
- **수평/수직 중앙 정렬의 공식**: 카드를 중앙에 배치할 때는 반드시 1) `min-h-screen`, 2) `flex-direction: column`, 3) `items-center/justify-center`, 4) **`max-width` 고정**이라는 4요소가 일관되게 적용되었는지 확인해야 합니다. 하나라도 빠지면 고해상도에서 화면이 늘어지는 현상이 발생합니다.
- **시각적 리듬의 규칙 (Visual Rhythm Rules)**: 데이터 중심의 대시보드에서는 복잡한 반응형 그리드(`grid-cols-*`)보다 세로 정렬이나 명확한 섹션 간격(`space-y-6`, `gap-6`)을 고정적으로 유지하는 것이 더 안정적인 UX를 제공합니다. 요소들의 '중력'이 한 방향으로 흐르게 하여 시선 분산을 최소화하세요.
- **Flex-First 입력창 데코레이션**: 입력창 내부에 아이콘을 배치할 때 `absolute` + `padding-left` 방식은 브라우저 렌더링이나 스타일 우선순위에 따라 글자가 아이콘 위로 겹치는 현상을 유발하기 쉽습니다. 대신 컨테이너에 `flex`와 `gap`을 주어 아이콘과 `input`을 물리적으로 나란히 배치하는 것이 레이아웃 깨짐을 방지하는 가장 확실한 방법입니다.
- **검색창 정렬의 황금 패턴**: 검색창에서 아이콘과 텍스트가 정렬되지 않는 문제는 매우 자주 발생합니다. 해결책은 **이중 컨테이너 구조**입니다: 외부 컨테이너(`rounded-xl overflow-hidden`)는 테두리와 모서리를 담당하고, 내부 컨테이너(`flex items-center px-4 gap-3 h-10`)는 실제 정렬을 담당합니다. `input`에는 인라인 스타일이나 복잡한 패딩을 주지 말고 `text-[14px]`만 지정하세요.
- **컴포넌트 수준의 패딩 오버라이드 (Component Padding Awareness)**: `Card`와 같은 공통 컴포넌트 사용 시 내부적으로 기본 패딩(`p-6` 등)이 설정되어 있다면, 외부에서 `className="p-0"`을 넘겨도 우선순위나 구현 방식에 따라 무시될 수 있습니다. 이 경우 컴포넌트가 지원하는 전용 prop(예: `padding="none"`)을 명시적으로 사용하여 '숨은 여백'을 먼저 제거해야 합니다.
- **데이터 밀도 최적화 (Granular UI Scaling)**: 관리자 도구처럼 정보 밀도가 중요한 화면에서는 1px, 2px 단위의 미세 조정(`py-2` -> `py-2.5`)이 사용자 경험에 큰 영향을 미칩니다. 사용자의 '너무 크다' 또는 '너무 타이트하다'는 피드백은 폰트 크기, 내부 여백, 외부 간격을 순차적으로 튜닝하여 최적의 'Compact' 지점을 찾아야 합니다.
- **인라인 편집 UI 패턴**: 관리자 페이지에서 항목을 수정할 때 별도의 모달을 띄우는 대신, 해당 카드 바로 아래에 편집 영역을 확장하는 인라인 방식이 더 직관적입니다. 이때 상단 카드의 `rounded-b-none`, `border-b-0`와 하단 편집 영역의 `border-t-0`를 조합하여 물리적으로 하나의 유닛처럼 보이게 만들어야 합니다.
- **콤마(Comma) 구분 입력창의 상태 관리**: 배열 데이터를 콤마로 구분된 문자열로 편집할 때, `join(', ')`된 값을 직접 `input`의 `value`로 쓰면 콤마 입력 시 즉시 `split`되어 입력이 차단되는 문제가 발생합니다. 반드시 `rolesInput`과 같은 **임시 문자열 상태**를 두고, 편집 시작(`onFocus` 또는 `Edit Button Click`) 시 초기화한 뒤 **저장 시점에만 배열로 변환**해야 합니다.
- **입력 필드 일관성 원칙**: 같은 페이지 내의 모든 입력 필드(검색창, 텍스트 입력 등)는 동일한 높이(예: `h-10`)를 유지해야 시각적 리듬감이 생깁니다. 여러 줄이 필요 없는 짧은 입력은 `textarea` 대신 `input`을 사용하는 것이 높이 제어가 쉽고 통일감을 줍니다.


- **인터랙티브 컴포넌트의 물리적 고립(Interactive Isolation)**: 팝업, 드롭다운 등 복잡한 상태를 가진 UI는 CSS(`opacity`, `visibility`)만으로 제어하기보다 React의 조건부 렌더링(`{isOpen && ...}`)을 통해 물리적으로 요소를 추가/제거하는 것이 가장 확실합니다. 이는 전역 전이(Global transitions)나 예상치 못한 CSS 간섭을 원천 차단합니다.




## 기술적 팁 (Technical Tips)
- **Fetch Wrapper & Credentials**: `fetch` 호출 시 `credentials: 'include'`를 명시하여 세션 쿠키가 백엔드로 전달되도록 함.
- **Graceful Auth Check**: 첫 세션 확인(`auth/me`) 시 401 에러를 예외로 던지지 않고 가공된 객체를 반환하여 불필요한 콘솔 에러 스트레스를 줄임.
- **FastAPI 상세 에러 연동**: 백엔드 에러 응답의 `detail` 필드를 우선적으로 파싱하여 사용자/개발자에게 구체적인 실패 원인을 제공함.
- **FastAPI 응답 검증(Validation)**: Pydantic 스키마에서 `str`로 정의된 필드에 `datetime` 객체를 반환하면 검증 에러가 발생합니다. 스키마 타입을 `datetime`으로 정의하면 자동 직렬화가 지원됩니다.
- **Path Aliases**: `tsconfig.json`의 `paths` 설정을 통해 `@/` 접두사를 사용하여 프로젝트 구조가 깊어져도 절대 경로로 임포트 가능하게 함.
- **Next.js API URL**: 클라이언트 사이드 호출을 위해 `NEXT_PUBLIC_` 접두사가 붙은 환경 변수를 사용하여 API URL을 관리함.
- **선언적 동선 제어(Declarative Flow)**: 미인증 유저에게 메인 페이지 대신 로그인 UI를 렌더링하는 방식은 리다이렉트(302)보다 부드러운 UX를 제공합니다. 단, 내부 컴포넌트들의 절대/상대 경로 임포트 이슈가 없는지 확인이 필요합니다.
- **이미지 기반 정렬 가이드**: 사용자가 이미지(예: Image 1 vs Image 2)로 피드백을 줄 때, 코드상의 수치보다 시각적 '덩어리'감에 집중하세요. 가로로 늘어지는 문제는 대부분 `max-width`의 부재에서 옵니다.
- **외부 클릭 감지(Click Outside Pattern)**: `useRef`와 `useEffect`를 조합하여 메뉴 외부 클릭을 감지하는 로직은 마우스 호버 방식보다 견고한 UX를 제공합니다. 이때 토글 버튼의 클릭 이벤트에서 `stopPropagation()`을 호출하여 버블링으로 인해 메뉴가 즉시 닫히는 현상을 방지해야 합니다.
- **레이아웃 정밀도 확보 (Precision Styling)**: 고정 폭을 가진 프리미엄 컴포넌트(드롭다운, 카드 등) 설계 시, 내부 텍스트가 짤리는 "Clipping" 현상을 방지하려면 컨테이너에 충분한 **내부 여백(Padding)**을 명시적으로 부여하고 `text-overflow: ellipsis`를 적절히 배합해야 합니다. 또한 전역 스타일 간섭을 피하기 위해 `z-index`는 2000 이상의 안전한 범위를 사용하세요.
- **상태 기반 스타일 강제 (Explicit State Styling)**: 내비게이션이나 사이드바의 `isActive` 상태에서 배경색이 크게 변할 경우(예: 짙은 핑크), 상속된 텍스트 색상에 의존하지 말고 `<span>` 태그 등으로 감싸 `text-white`를 명시적으로 할당해야 시인성을 확실히 보장할 수 있습니다.
- **Grid 레이아웃 명시의 중요성**: `grid`만 사용하면 예상치 못한 열 배치(카드와 편집 영역이 나란히 배치되는 등)가 발생할 수 있습니다. 세로 스택을 의도한다면 반드시 `grid-cols-1`을 명시하고, 자식 요소에 `w-full`을 추가하여 레이아웃을 강제해야 합니다.
- **UI 간격 조정의 단계적 접근**: 사용자가 간격에 대해 불만을 표현할 때는 한 번에 큰 변화를 주기보다 `space-y-4` → `space-y-5` → `space-y-6` 같은 점진적 조정을 통해 최적점을 찾는 것이 효율적입니다. 특히 버튼과 입력창 사이의 시각적 분리는 `pt-3`보다 `pt-5` 정도가 명확한 구분감을 줍니다.
- **카드 패딩 일관성 원칙**: 같은 프로젝트 내의 모든 리스트형 카드는 동일한 내부 패딩(`px-4 py-2.5`)을 유지해야 합니다. 새로운 페이지를 만들 때 기존 페이지의 카드 스타일을 정확히 복사하여 시각적 일관성을 유지하세요. "비슷하게" 만드는 것이 아니라 "정확히 동일하게" 만들어야 합니다.
- **작업 전 파일 경로 확인 필수**: 사용자가 "권한 관리"라고 요청했을 때, `/admin/permissions`와 `/admin/role-rules` 중 어느 것인지 **반드시 먼저 확인**해야 합니다. 비슷한 이름의 페이지가 여러 개 있을 때는 `grep_search`로 제목을 검색하거나 사용자에게 명확히 물어봐야 합니다. 잘못된 파일을 수정하면 모든 작업이 무효화됩니다.
- **서버 리다이렉트로 인한 CORS 차단**: URL 끝에 슬래시(`/`)가 빠지면 백엔드가 307 Redirect를 보내면서 CORS 헤더가 유실될 수 있습니다. DB 업데이트는 성공하지만 프론트엔드가 결과를 못 읽는다면 엔드포인트에 슬래시를 추가하거나, 서버의 `TrailingSlash` 설정을 확인해야 합니다.






## 실패 및 해결책 (Failures & Fixes)
- **TypeScript Window Type**: HTML `dialog.showModal()` 등을 호출할 때 TypeScript에서 타입을 인식하지 못할 경우 `(window as any).modalId.showModal()`로 우회하거나 글로벌 타입 정의가 필요함.
- **OAuth Redirect URI Mismatch**: 구글 클라우드 콘솔의 '승인된 리디렉션 URI'는 `localhost`와 `127.0.0.1`을 다르게 인식하므로 브라우저 접속 주소와 정확히 일치시켜야 함.
- **CORS & Credentials**: 프론트엔드에서 `credentials: 'include'`를 사용할 경우 백엔드 CORS 설정에서 `allow_credentials=True`와 구체적인 `origins` 리스트 설정이 필수임 (`*` 사용 불가).
- **Backend 500 & CORS Fix**: `/admin/role-rules`에서 발생한 500 에러는 백엔드 스키마에서 날짜 형식이 문자열 정적 타입으로 지정되어 발생한 데이터 타입 불일치 문제였습니다. 백엔드 코드 수정이 필요함을 식별하였습니다.
- **약관 동의 무한 루프**: 약관 동의 후 DB에는 저장되었으나 프론트엔드 유저 객체가 갱신되지 않아 다시 `/terms`로 리다이렉트되는 이슈가 있었습니다. 제출 성공 직후 `auth/me`를 다시 호출하여 컨텍스트를 동기화함으로써 해결했습니다.
- **고정 배경 위 텍스트 시인성**: Rose-600과 같은 진한 강조색 배경 위에서는 상속된 어두운 텍스트가 묻힐 수 있습니다. 활성 상태(Active)에 따라 `text-white`와 같은 명시적 반전 색상을 강제로 적용해야 합니다.
- **대시보드 그리드 간격 불일치**: 카드 수에 따라 가변적인 그리드를 사용할 때, 요소 내부의 데이터 양이 다르면 시각적 균형이 깨집니다. 이 경우 섹션별로 고정된 간격(`gap`)을 부여하고 단일 열(Single-column) 기반의 리스트형 배치를 활용하여 리듬감을 유지함으로써 해결했습니다.
- **검색창 아이콘 겹침 이슈**: `pl-14`와 같은 패딩 설정이 무시되거나 placeholder와 아이콘이 겹치는 현상은 `flexbox` 구조(아이콘과 input을 형제 요소로 분리)로 전환하여 원천 차단했습니다.
- **검색창 정렬 반복 실패**: 검색창에서 아이콘과 텍스트 정렬이 깨지는 문제가 반복적으로 발생했습니다. 근본 원인은 단일 컨테이너에 `border`, `rounded`, `flex`를 모두 적용하면서 `input`에 인라인 스타일을 추가하는 복잡한 구조였습니다. **이중 컨테이너 패턴**(외부: 테두리/모서리, 내부: flex 정렬)을 사용하고 `input`은 최소한의 스타일만 적용하여 해결했습니다.
- **카드 내부 여백 축소 실패**: `className="p-0"`을 적용해도 여백이 사라지지 않을 때, 상위 컴포넌트(`Card`)의 기본 `padding` prop을 `none`으로 설정하여 스타일 충돌을 해결했습니다.
- **인라인 편집 영역 레이아웃 깨짐**: 카드와 편집 영역이 의도와 달리 나란히(side-by-side) 배치되는 문제가 발생했습니다. 부모 컨테이너에 `grid-cols-1`을 명시하고, 자식 요소들에 `w-full`을 추가하여 세로 스택을 강제함으로써 해결했습니다.
- **프로필 이미지 크기 조정 반복**: 사용자의 "너무 크다"/"너무 작다" 피드백에 따라 `w-9` → `w-11` → `w-10`으로 여러 번 조정했습니다. 초기에 사용자의 선호도를 더 명확히 파악하거나, 중간값부터 시작하는 것이 반복을 줄이는 방법입니다.
- **잘못된 파일 수정으로 인한 전체 실패**: 사용자가 "권한 관리" 페이지 수정을 요청했는데, `/admin/role-rules/page.tsx`를 수정하여 모든 변경사항이 적용되지 않았습니다. 실제 파일은 `/admin/permissions/page.tsx`였습니다. **교훈**: 작업 시작 전에 반드시 `grep_search`로 제목이나 특정 텍스트를 검색하여 정확한 파일 경로를 확인해야 합니다. 추측으로 파일을 선택하면 안 됩니다.



- **내비게이션 메뉴 먹통 및 정렬 이슈**: Tailwind CSS의 `group-hover`와 React 상태가 충돌하여 메뉴가 항상 보이거나 토글되지 않던 문제는 인라인 스타일과 명시적 상태 제어로 해결했습니다. 또한 정보 출력부의 `padding` 부족으로 텍스트가 짤리는 현상은 고정 폭(`width`)과 내부 여백의 명시적 정의로 해결하였습니다.


## 새로운 디자인 패턴
- **프리미엄 로그인 경험**: 메쉬 그레이디언트 배경과 글래스모피즘 카드의 조합은 서비스의 첫인상을 결정짓는 강력한 시각적 장치입니다.
- **상태 기반 마이크로 인터랙션**: 버튼의 Hover 시 미세한 Y축 이동(-2px)과 그림자 변화는 사용자에게 명확한 피드백을 제공합니다.
